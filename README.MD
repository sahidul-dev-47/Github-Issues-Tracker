## 📖 About The Project

**GitHub Issues Tracker** is a front-end web application that allows users to create, track, and manage project issues — similar to GitHub's native issue system. Users can open new issues, assign them to team members, set priority levels, add labels like `BUG` and `HELP WANTED`, and close issues once resolved.

---

## ✨ Features

- ✅ **Create New Issues** — Add issues with title, description, labels, and assignee
- 🔍 **Filter Issues** — View All, Open, or Closed issues instantly
- 🏷️ **Labels Support** — Tag issues as `BUG`, `HELP WANTED`, and more
- 👤 **Assignee Management** — Assign issues to specific team members
- 🚨 **Priority Levels** — Set High, Medium, or Low priority per issue
- 📅 **Date Tracking** — See when each issue was opened
- ❌ **Close Issues** — Mark issues as resolved with one click
- 📱 **Responsive Design** — Works across all screen sizes

## 💡 How It Works

1. Click **+ New Issue** to open the issue creation form
2. Fill in the title, description, labels, assignee, and priority
3. Submit to add it to the issue list
4. Use the **All / Open / Closed** tabs to filter issues
5. Click **Close** on any issue to mark it as resolved
